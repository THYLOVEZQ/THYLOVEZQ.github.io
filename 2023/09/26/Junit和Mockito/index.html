Junit声明周期方法

  • 当我们创建测试用例时，我们可能需要进行一些常见的操作。
  • 在每次测试时
    • 创建对象，设置测试数据
  • 在每次测试后
    • 释放资源，清除测试数据

Junit提供了声明周期方法注释：

注解 描述
@BeforeEach 在每个测试方法前执行，在编写相同代码时使用：
例如创建对象，设置测试数据
@AfterEach 在每个测试方法后执行：
例如释放资源，清理测试数据
… …

Mockito

主项目架构

我们怎么测试服务，我们只想用最小的依赖配置来测试服务。

我们可以使用Dao对象的替身

运用替身的技术被称为“Mocking”。

Mocking的优点

  • 允许我们去独立测试一个类
  • 测试所给定类和其依赖之间的联系
  • 减少配置/使用依赖
  • 例如在DAO，DB，REST API 等
    • 我们可以mock DAO来给出响应
    • 我们可以mock 一个REST API 去给出响应

Mocking Frameworks

  • Java生态系统包含一系列的Mocking框架
  • 这些Mocking框架提供了下述功能：
    • 减少Mock的硬编码
    • 设置Mock响应的期望
    • 验证调用方法包括调用次数
    • 支持程序抛出异常

Mock框架
